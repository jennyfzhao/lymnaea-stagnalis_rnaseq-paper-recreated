# Software and Data Workflow Table

This table lists the software and data sources that were actually used in the working
*Lymnaea stagnalis* CNS aging RNA-seq reproduction workflow.

| Order | Software / resource used | What it does | Biological purpose in this project | Where it came from / how it was installed | Command or command pattern used | Data source and input style | Output style and project location |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | NCBI SRA metadata and run accession lists | Defines which public sequencing runs belong to the project. | Links each RNA-seq sample to the original public BioProject, age group, and replicate structure before any analysis. | Data were obtained from NCBI BioProject `PRJNA698985`; local metadata files are stored in `software/`. | Metadata files used directly: `PRJNA698985_runinfo.csv`, `PRJNA698985_SraAccList.txt`, and related accession-list files. | CSV/TSV/text metadata from NCBI SRA; key biological fields include paired-end RNA-seq layout, Illumina NovaSeq 6000 platform, and run accessions. | Sample/run metadata in `software/` and `data/sample_metadata.tsv`. |
| 2 | SRA Toolkit `prefetch` and `fasterq-dump` v3.4.1 | Downloads SRA archives and converts them to FASTQ. | Turns public archived reads into paired FASTQ files that can be quality-checked and aligned. | Install with `environment.yml` from bioconda, or from NCBI SRA Toolkit binaries: `https://github.com/ncbi/sra-tools/wiki/02.-Installing-SRA-Toolkit`. | `JOBS=3 THREADS_PER_JOB=4 software/download_fastq_parallel.sh software/PRJNA698985_SraAccList.txt`; inside the script: `prefetch "$accession" --output-directory "$SRA_DIR"` and `fasterq-dump "$SRA_DIR/$accession" --outdir "$FASTQ_DIR" --temp "$run_tmp" --split-files --skip-technical --threads "$THREADS_PER_JOB" --progress`. | Input: SRA accession IDs such as `SRR13618140`; output source data are paired-end RNA-seq reads. | `.sra` archives in `data/sra/`; paired FASTQ files such as `SRR13618140_1.fastq` and `SRR13618140_2.fastq` in `data/fastq/` (excluded from Git because they are large). |
| 3 | FastQC v0.12.1 with Java 21 | Generates per-read quality-control reports from FASTQ files. | Checks whether sequencing reads have quality, adapter, GC-content, or overrepresented-sequence issues before interpreting alignments. | Install with `environment.yml` from bioconda, or from Babraham Bioinformatics: `https://www.bioinformatics.babraham.ac.uk/projects/fastqc/`. | Example command used in project notes: `fastqc data/fastq/SRR13618150_1.fastq data/fastq/SRR13618150_2.fastq --outdir data/fastqc-results --threads 2`. | Input: paired FASTQ files. | HTML and ZIP reports in `data/fastqc-results/` (excluded from Git in the portfolio repo). |
| 4 | NCBI *L. stagnalis* reference genome `GCA_900036025.1` | Provides the genome sequence used for read alignment. | Gives HISAT2 a reference coordinate system for mapping CNS RNA-seq reads to the snail genome. | Downloaded from NCBI Assembly/GenBank; local file stored under `data/reference_genome/`. | Reference was used as the input to `hisat2-build`. | Input style: genome FASTA, e.g. `GCA_900036025.1_v1.0_genomic.fna`. | Reference FASTA in `data/reference_genome/` and HISAT2 index in `reference/` (both excluded from Git because of size). |
| 5 | HISAT2 v2.2.1 | Builds a splice-aware genome index and aligns paired RNA-seq reads to the genome. | Maps reads to the genome so transcript assembly and expression quantification can be performed. | Install with `environment.yml` from bioconda, or from HISAT2 releases: `https://github.com/DaehwanKimLab/hisat2/releases`. | Index command pattern: `hisat2-build data/reference_genome/GCA_900036025.1_v1.0_genomic.fna reference/hisat2_i`; alignment command recorded from SAM headers: `hisat2 -x reference/hisat2_i -S data/aligned/SRR13618140.sam --threads 4 --summary-file data/aligned/logs/SRR13618140_hisat2_summary.txt -1 data/fastq/SRR13618140_1.fastq -2 data/fastq/SRR13618140_2.fastq`. | Input: paired FASTQ files and genome index prefix `reference/hisat2_i`. | SAM alignment files in `data/aligned/`, later converted to sorted BAM files; alignment files are excluded from Git because they are large. |
| 6 | Samtools v1.23.1 | Converts, sorts, and indexes alignment files. | Produces coordinate-sorted BAM files required by StringTie and other downstream alignment-based tools. | Install with `environment.yml` from bioconda, or from HTSlib/Samtools downloads: `https://www.htslib.org/download/`. | Command pattern used: `samtools sort -@ 4 -o data/aligned/SRR13618140.sorted.bam data/aligned/SRR13618140.sam`; indexing pattern: `samtools index data/aligned/SRR13618140.sorted.bam`. | Input: SAM alignment files from HISAT2. | Sorted BAM and BAI files in `data/aligned/` (excluded from Git). |
| 7 | StringTie v3.0.3 | Assembles transcripts and estimates transcript/gene abundance from sorted BAM alignments. | Reconstructs expressed transcript models and produces transcript/gene abundance values for CNS samples. | Install with `environment.yml` from bioconda, or from StringTie releases: `https://github.com/gpertea/stringtie/releases`. | Assembly/quantification commands are recorded in GTF headers, for example: `stringtie data/aligned/SRR13618140.sorted.bam -p 4 -G data/stringtie/stringtie_merged.gtf -e -B -o data/stringtie/quant/SRR13618140/SRR13618140.gtf -A data/stringtie/quant/SRR13618140/SRR13618140.gene_abund.tsv`; merge command recorded in `stringtie_merged.gtf`: `stringtie --merge -p 4 -o data/stringtie/stringtie_merged.gtf data/stringtie/assemblies/mergelist.txt`. | Input: sorted BAM files and merged/reference GTF where applicable. | Per-sample GTF files and gene abundance TSV files under `data/stringtie/`; merged GTF `data/stringtie/stringtie_merged.gtf`; excluded from Git because they are large/regenerable. |
| 8 | gffread v0.12.9 | Extracts transcript sequences from a transcript annotation and genome FASTA. | Converts the merged transcript annotation into transcript FASTA input for coding-sequence prediction. | Install with `environment.yml` from bioconda; source: `https://github.com/gpertea/gffread`. | `gffread data/stringtie/stringtie_merged.gtf -g data/reference_genome/GCA_900036025.1_v1.0_genomic.fna.gz -w data/annotation/transcripts.fa`. | Input: merged StringTie GTF plus genome FASTA. | Transcript FASTA `data/annotation/transcripts.fa` (excluded from Git because of size). |
| 9 | TransDecoder v6.0.0 | Predicts likely coding regions and protein sequences from transcript FASTA. | Produces predicted *L. stagnalis* ORFs, CDS FASTA, and protein FASTA for downstream protein annotation. | Install with `environment.yml` from bioconda, or from TransDecoder releases: `https://github.com/TransDecoder/TransDecoder/releases`. | `TransDecoder.LongOrfs -t data/annotation/transcripts.fa -O data/annotation`; final prediction/integration command: `TransDecoder.Predict -t data/annotation/transcripts.fa -O data/annotation --retain_blastp_hits data/annotation/blastp_refseq9/lymnaea_transdecoder_vs_refseq9.blastp.tsv --retain_pfam_hits data/annotation/pfam/transdecoder_pfam.domtblout`. | Input: transcript FASTA; later BLASTP and Pfam evidence files. | Final `.pep`, `.cds`, `.gff3`, and `.bed` outputs in `data/annotation/`; large FASTA/GFF outputs are excluded from Git, while compact downstream annotation tables are tracked. |
| 10 | NCBI BLAST+ `blastp` v2.17.0+ and NCBI `taxdb` | Searches predicted proteins against NCBI RefSeq proteins. | Adds homology evidence from related molluscan species to support predicted ORFs and biological interpretation. | Install BLAST+ with `environment.yml` from bioconda, or NCBI binaries: `https://ftp.ncbi.nlm.nih.gov/blast/executables/blast+/LATEST/`; download `taxdb` from `https://ftp.ncbi.nlm.nih.gov/blast/db/taxdb.tar.gz`. | `blastp -query "$QUERY_PEP" -remote -db refseq_protein -entrez_query "$REFSEQ9_QUERY" -evalue 1e-3 -word_size 6 -max_target_seqs 20 -outfmt "6 qseqid sseqid staxids sscinames pident length mismatch gapopen qstart qend sstart send evalue bitscore stitle" -out "$OUTDIR/lymnaea_transdecoder_vs_refseq9.blastp.tsv"`. | Input: TransDecoder predicted protein FASTA; remote NCBI `refseq_protein` database filtered to nine species related to *L. stagnalis*. | Tabular BLASTP TSV in `data/annotation/blastp_refseq9/lymnaea_transdecoder_vs_refseq9.blastp.tsv`. |
| 11 | HMMER3 `hmmscan` and Pfam-A | Searches predicted proteins for conserved Pfam domains. | Identifies protein domains that help interpret predicted ORFs and differentially expressed genes. | Install HMMER with `environment.yml` from bioconda, or from `http://eddylab.org/software/hmmer/`; download Pfam-A from `https://ftp.ebi.ac.uk/pub/databases/Pfam/current_release/Pfam-A.hmm.gz`. | Command recorded in the domtblout header: `hmmscan --domtblout data/annotation/pfam/transdecoder_pfam.domtblout --cpu 8 data/annotation/pfam/Pfam-A.hmm transcripts.fa.transdecoder_dir/longest_orfs.pep`. | Input: TransDecoder longest ORF protein FASTA and Pfam-A HMM database. | HMMER domain table `transdecoder_pfam.domtblout` plus summarized Pfam tables produced later; raw HMM database and domtblout are excluded from Git because of size. |
| 12 | Python 3.14.5 custom script `build_pfam_annotation_tables.py` | Parses Pfam `domtblout` results into protein-level and gene-level annotation tables and joins Pfam annotations to DESeq2 outputs. | Makes domain annotation usable for biological interpretation of differential expression results. | Custom script in `scripts/`; run with the project Python environment. | `python scripts/build_pfam_annotation_tables.py --pfam-domtblout data/annotation/pfam/transdecoder_pfam.domtblout --deseq-dir figures/deseq2_results --output-dir data/annotation/pfam`. | Input: HMMER/Pfam domtblout and DESeq2 CSV files. | `protein_pfam_domains.tsv`, `gene_pfam_summary.tsv`, and `data/annotation/pfam/deseq2_annotated/*_with_pfam.csv`. |
| 13 | R with DESeq2, ggplot2, and pheatmap | Performs differential expression analysis and creates PCA, volcano, and heatmap figures. | Tests which genes change across 3-month, 6-month, and 18-month CNS samples and visualizes sample/gene expression patterns. | Install with `environment.yml` from conda-forge/bioconda, or from CRAN/Bioconductor. | `Rscript figures/rnaseq_deseq2_figures.R`. | Input: gene count matrix `data/expression_matrices/gene_count_matrix.csv` and metadata `data/sample_metadata.tsv`. | DESeq2 result CSVs in `figures/deseq2_results/`; PCA, volcano, and heatmap PNG/PDF files in `figures/`. |
| 14 | Python 3.14.5 custom script `make_annotated_de_plots.py` | Builds plots and summary tables from Pfam-annotated DESeq2 results. | Highlights significant differentially expressed genes with interpretable Pfam domain context. | Custom script in `scripts/`; run with the project Python environment. | `python scripts/make_annotated_de_plots.py --input data/annotation/pfam/deseq2_annotated/18_month_vs_3_month_DESeq2_results_with_pfam.csv --output-dir figures/annotation_plots/18_month_vs_3_month`. | Input: Pfam-annotated DESeq2 CSV for the 18-month vs 3-month comparison. | Annotated volcano plot, top Pfam-domain plot, top annotated gene table, and summary file in `figures/annotation_plots/18_month_vs_3_month/`. |
| 15 | Pfam2GO mapping file from Gene Ontology | Maps Pfam accessions to GO terms. | Provides GO annotation from Pfam domain results. | Downloaded from the Gene Ontology `external2go` mapping source and stored as `data/annotation/go_pfam2go/pfam2go`. | `curl -L https://current.geneontology.org/ontology/external2go/pfam2go -o "$GO_DIR/pfam2go"`; then `awk` joins mapped Pfam IDs from `protein_to_pfam.tsv` to GO terms. | Input: `protein_to_pfam.tsv` derived from HMMER/Pfam output and the `pfam2go` mapping file. | `data/annotation/go_pfam2go/lymnaea_protein_go_pfam2go.tsv` and `data/annotation/enrichment/gene_to_go.tsv`. |
| 16 | KofamScan with KEGG KOfam profiles | Assigns KEGG Orthology identifiers using profile HMMs. | Provides KEGG KO annotation for predicted proteins. | KofamScan was cloned from GitHub into `software/kofam_scan`; `ko_list.gz` and `profiles.tar.gz` were downloaded from KEGG/KOfam into `data/annotation/kofam/`; GNU `parallel` and HMMER were available in the annotation environment. | `ruby "$SNAIL/software/kofam_scan/exec_annotation" -o "$KOFAM_DIR/lymnaea_kofam.mapper.tsv" -f mapper -p "$KOFAM_DIR/profiles" -k "$KOFAM_DIR/ko_list" --cpu 8 --tmp-dir "$KOFAM_DIR/tmp" "$PEP"`. | Input: TransDecoder predicted protein FASTA and KOfam HMM profile database. | KEGG Mapper-compatible TSV `data/annotation/kofam/lymnaea_kofam.mapper.tsv` and gene-level table `data/annotation/enrichment/gene_to_kegg_ko.tsv`; large KOfam profile files are excluded from Git. |
| 17 | Base R enrichment scripts | Runs Fisher exact tests and FDR correction for GO terms and KEGG KO terms. | Tests whether annotated functions are overrepresented among differentially expressed genes. | Custom scripts in `data/annotation/enrichment/`; base R version was used because the first tidyverse script required unavailable packages. | `Rscript data/annotation/enrichment/run_go_kegg_enrichment_baseR.R`; plotting command: `Rscript data/annotation/enrichment/plot_enrichment_baseR.R`. | Input: DESeq2 CSVs, gene-to-GO table, and gene-to-KEGG-KO table. | Enrichment CSV files and annotated DESeq2 CSV files in `data/annotation/enrichment/`; selected GO and exploratory KEGG KO figures in `figures/enrichment/`. |
